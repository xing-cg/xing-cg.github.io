函数参数默认值

总结

因为函数参数的默认值是在编译器带入的，所以函数的参数的默认值只能在本单文件（不包括头文件，头文件的信息是在预编译期就展开了。）生效。

inline

在调用内联函数时，该函数会在调用点将代码展开（复制）。

与一般函数的对比

一般函数调用：

  1. 传参
  2. call
  3. 开辟栈帧
  4. 返回值返回
  5. 栈帧回退
  6. 参数清除

内联函数：

  1. 在函数的调用点展开（函数体中的指令复制过去）
  2. 不用传参
  3. 不用call
  4. 不用…

debug和release

在debug版本，内联函数和正常函数调用方式一致；

在release版本，在调用内联函数时候，该函数会在调用点展开。

哪些函数不能被展开

提示：内联函数是在编译时期，生成指令时展开的。

答案是递归函数无法展开。因为递归函数的终止条件一定需要由变量决定，递归层数不定。而编译期是无法得知变量的具体值。所以递归函数不能被处理为内联函数。

为何不会报错。

inline只是对系统的建议，声明建议将该函数处理为内联。实际情况由编译器视情况决定。

符号

符号=数据+指令

符号的来源

所有的数据都会生成符号。

指令中（比如函数调用）只有函数名会生成符号。

符号又可分为两种

全局符号-global 符号

所有的文件只要引用声明就可以使用。

局部符号-local 符号

只有本文件可见。

普通的函数生成的是global符号。但是，被inline修饰的话，只在本文件可见，因为生成的是local符号。

如果函数只是声明，生成的符号是UND。

inline函数在debug版本生成的是local型符号，如果处理为内联之后，在release版本下不生成符号，因为它已在调用点处展开了。

宏函数 static函数 内联函数

宏函数

  1. 预编译时期在调用点展开
  2. 无法调试
  3. 没有参数类型安全校验
  4. 单文件可见
  5. 不生成符号

static函数

  1. 不展开
  2. 可以调试
  3. 有参数类型校验
  4. 单文件可见
  5. 生成local型符号

内联函数

  1. debug版本不展开，可以调试；release版本在调用点展开，不可调试
  2. 有参数类型校验
  3. 单文件可见
  4. debug生成local符号；release版本不生产符号。
    • 为何在release版本不生成符号？：都已经展开了，没必要生成符号以标识
    • 为何在debug版本生成符号？：debug是为了未来的release，既然要展开，就要知道具体的函数定义在哪里，所以要生成符号以标识。

普通函数

可以调试，有参数类型校验，多文件可见，生成global符号。

函数的重载

函数的重载–静多态–编译时期的多态–早绑定

函数的重载就要看准函数的原型：包括函数返回类型、函数名、形参列表（其中形参名可省略），且不需要函数体。

但是我们不能拿函数返回类型作为重载的标志，因为会产生二义性。

Cpp中的重载就是：函数名相同，参数列表不同。

符号不能同名，否则被认为是重定义。因此生成的符号不能一样。

C语言生成函数符号依赖函数名。

C++生成函数符号依赖函数名和参数列表（返回值不影响）。

函数重载是在编译时期决定的究竟调用哪个函数，因为编译生成指令和符号，才能确定call哪个具体的函数——静多态的一种（暂时先记住，后面会一直说，因为面试一直面

C：常-变量

不能作为左值。

C++：常量

面试

为什么常量必须初始化？

因为如果要使用常量又不初始化的话，后期没有机会改。使用一个随机值对于程序没有意义。

如果使用变量给const修饰的量初始化，则该量会退化为常变量。

指针和const

要点1：const修饰的内容不能作为左值

要点2：不能泄露常量的地址给非常量的指针

const修饰的类型是离它最近的第一个成型的类型。其余的是它修饰的内容。

如果const修饰的内容不包含指针，则无法参与类型。

new和malloc区别

  1. new是关键字，malloc是函数，new调用malloc。
  2. new和malloc都是在堆区申请空间
  3. new有三个步骤
    1. 开辟空间
    2. 构造函数。—初始化，这是与malloc的区别。
    3. 返回地址
  4. new不会强转返回值类型
  5. 封装计算sizeof
  6. 空间不足时，new会抛异常（bad_alloc）。malloc会返回空指针。（int *p = new(nothrow) int(20);）
