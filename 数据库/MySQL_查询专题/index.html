语法

limit M,N - limit N

limit只是改变了数据显示的效果吗？对查询的效率有没有提升呢？先引入一个关键字explain。

explain是一个关键字，是定位SQL、索引性能的有利工具。

使用它加载SQL语句开头，可以查看SQL语句的执行计划。

explain展示的字段：

  1. id: 编号
  2. select_type: 如SIMPLE，表示查询语句的复杂程度类型
  3. table: 如user，当前表
  4. partitions: NULL
  5. type: const
  6. possible_keys: name，可能用到的索引
  7. key: name，实际用到的索引
  8. key_len: 152
  9. ref: const
  10. rows: 1，查询语句可能需要扫描的行数。
  11. filtered: 100.00
  12. Extra: NULL
